前言

POST请求的请求体格式

使用代码时在content-type中指定，使用接口工具，选中POST时也有对应的类型选择

none

不传递请求体，类似于get请求。通常会用于资源获取，状态查询之类

application/x-www-form-urlencoded

数据以键值对的形式，就像是URL参数一样，特殊字符也会被转义，比如空格会转义为+或者%20（不同标准中，但都能被识别）

  • 适用于简单的表单提交

application/json

数据以json对象的形式

  • 自由的数据格式

application/octet-stream

直接发送二进制数据，对于图片这种还有image/jpeg、image/png等

  • 适用于文件上传

multipart/form-data

支持上传文件和二进制数据，同时也可以包含文本字段。请求体会被分为多个部分，每个部分都有自己的Content-Type

  • 适用于上传文件、含文本和文件的表单提交

原生

XMLHttpRequest

最早的ajax实现

注意：XHR 有几个常用但容易被忽视的坑：

fetch

现代ajax的实现，现代浏览器提供的 Promise-based API。支持更多HTTP功能，如请求头设置、超时设置、请求取消等等等等等

链式调用

async/await

为什么要response.json()？

fetch返回的是一个Response对象，Response对象中包含了如状态码、状态信息、响应头、响应体等信息，这个方法是将响应对象转换为json对象

流式数据处理（如大文件下载），边接收边处理

axios

基于Promise，浏览器端基于XHR，node端基于node内置http模块。

优势在于拦截器、自动转换JSON数据（fetch需要res.json()）、内置上传进度、取消请求等

请求拦截

响应拦截

请求取消（CancelToken），比原生的fetch取消更加早，axios在2014年支持请求取消，fetch在2017年支持

请求取消（AbortController），向fetch原生靠拢

got

Node.js 专用 HTTP 客户端，比 axios 更现代、更强大的 API 设计。

核心优势

1. 流式处理（大文件下载）

2. 自动重试（retry）

3. 内置缓存（相当于做个代理）

4. JSON 自动解析 + 请求体自动序列化

5. 强大的错误处理

6. 链式调用（搜索参数、超时、头设置）

got vs axios 对比

特性 got axios
环境 Node.js 专用 浏览器 + Node
流式支持 原生 .stream() 需 responseType: 'stream'
重试 内置，配置丰富 需额外插件
缓存 内置 需插件
Promise API 天生 Promise 天生 Promise
请求取消 AbortController CancelToken（已废弃）

注意：got v12+ 全面支持 ESM，如果项目用 CommonJS 需要用 import got from 'got' 配合 .default，或者直接用 create 实例方式。

安全相关

token 存储位置

方式 优点 缺点
localStorage 刷新不丢失 XSS 可以被读取
sessionStorage 页面关闭自动清除 刷新会丢失
cookie（HttpOnly） JS 读不到，防 XSS 依赖 cookie 策略

建议：AccessToken 放 localStorage，敏感操作加短信/密码二次验证；RefreshToken 用 HttpOnly cookie

CSRF 防御

攻击原理：浏览器自动携带 cookie，诱导用户访问恶意站点发起跨域请求。

防御手段：

  1. 请求头加上 Origin / Referer 校验
  2. 接口要求 withCredentials: true
  3. 请求带上 CSRF Token（cookie 中设置，header 中携带）
