React 和 JSX

jsx 是 JavaScript XML 的简写，在 JavaScript 文件中写 HTML-like 标签

  • 因为jsx不是标准的ECMAScript语法，而是它的语法扩展，所以在普通环境是不可以使用的；要在脚手架中使用（因为脚手架中包含了babel，@babel/preset-react）

JSX 和 React 是两个东西，前者是语法拓展，后者是一个 JavaScript 库。React 是使用了这种语法拓展。通常是一起使用的，但也可以分开使用。更多介绍

语法规则

1. 单根节点

原因：JSX 看起来像 HTML，但是是转换成原生的 JavaScript 对象，一个方法是不可以返回两个对象的，所以才需要包裹起来

  • 这个空白的标签叫做 Fragment （片段 ？？）
  • 完整样子：<Fragment></Fragment>
  • 向 Fragment 传 key，不能使用短标签 <></>，需要从 react 导入 Fragment 然后使用 <Fragment key={yourKey}>...</Fragment>
  • 不会重置state，在从<><Child /></>到[<Child>]或反转的时候；<><><Child /></></>就会重置。关于重置 state，需要跳转看到state章节
  • class => className 、for => htmlFor，等等
  • 如果元素没有子节点，可以转为单标签：<span></span> => <span />，当然不转也行

3. 驼峰式属性名

更新 数组 类型的 state

React 中 state 中数组的操作方法推荐（避免使用改变数组的，推荐使用返回新数组的）：

避免 推荐
添加 push, unshift concat, […arr]
删除 pop, shift, splice filter, slice
替换 splice, arr[i]=xx map
排序 reverse, sort 先复制这个数组

添加内容：和对象一样，使用扩展运算符来复制以达到改变某一个值的目的

  • 要改变新插入值的位置，将新插入的值这行放到扩展运算的上面就行

删除内容：最简单的就是过滤掉这个不需要的，或者直接创建个新数组的不包含这个要删除的内容

替换内容：在原有的基础上创建一个新的数组，使用map，如果符合，改变这个值然后返回，不符合的返回原来的样子

插入内容：确定要添加的位置，然后使用slice分割数组，将要添加的放到两个切片中间

其他操作：比如 反转，排序，js 方法会改变旧的数组，所以要先复制出一个数组然后再做出改变，如nextList.sort()，nextList[0] = {name: "zs", age: 18}

对于数组 list 和 nextList，虽然不是相同的数组，但是list[0]和nextList[0]指向的是相同的对象，所以直接nextList[0].age=19这样的还是不推荐的，因为这是浅拷贝，是直接改掉了对象里面的东西

2️⃣ 推荐语法：setState( (state, props) => {} )

3️⃣ setState()的第二个参数，是一个回调函数

  • 如果希望更新后执行什么操作，就可以使用这个回调函数
  • 在 DOM 渲染后执行（和componentDidMount可以相互使用）

虚拟 DOM 和 Diff 算法

  • 只要state变化就重新渲染视图，有时候会浪费性能。解决这个问题，用到虚拟 DOM 和 Diff 算法
  • 虚拟 DOM 本质上就是一个 JavaScript 对象，用来描述希望看到的内容。（实际上就是 jsx 对象）

执行过程

  1. 初次渲染时，React 会根据初始 State。创建一个虚拟 DOM 对象（树）
  2. 根据虚拟 DOM 生成真正的 DOM 然后渲染到页面中
  3. 当数据变化后（setState()），重新根据新的数据，创建新的虚拟 DOM 对象
  4. 使用diff算法，找到与上一个虚拟 DOM 对比，然后渲染需要更新的内容
  5. 然后 React 只更新（patch）变化的内容，渲染到页面中

render 方法的调用并不意味着浏览器中的重新更新，仅仅说明要进行 diff

虚拟 DOM 不是真正的 DOM，只要可以运行 JavaScript 的地方就可以使用，这就使得 React 可以脱离浏览器而存在，可以在 Android 和 IOS 中使用

useEffectEvent

📦 React 19

在 Effect 里能安全地使用最新的 props 和 state，同时又不必把这些频繁变化的值放进依赖数组里。

场景

  • 不加 theme，主题切换时on回调里面的theme还是旧的值
  • 加上 theme，Effect 会重新执行导致connection断开重连

解决方式：

原理是使用了类似“代理模式”的机制：

  1. 返回的onConnected是一个稳定、不变的引用。意味着把它放在 Effect 的依赖数组中不会导致 Effect 重运行
  2. 内部保持更新：React 背后维护一个.current属性，指向传给useEffectEvent的最新函数逻辑，当调用onConnected时，React 调用.current指向的最新函数，从而确保能访问到最新的theme

常见使用场景：

  • 订阅和事件监听：WebSocket、EventEmitter 或者 DOM 事件回调函数需要访问状态，但是又不能因为状态变化而重新订阅（比如出发了重新渲染）
  • 定时器需要读取最新值，但是 Effect 又不能因为这个值的变化触发 Effect 而重新计算

使用限制：

  • 尽在 Effect 中调用
  • 如果 Effect 本身需要这个依赖才能触发，那不应该在useEffectEvent中使用

Profiler

用于测量 React 组件渲染的性能开销。它需要两个属性：id（字符串）和 onRender 回调（子组件树更新时会被调用）。

用途：开发环境识别渲染瓶颈，分析哪些组件重新渲染慢。

回调参数：id, phase, actualDuration, baseDuration, startTime, commitTime

  • id：<Profiler>树的 id 属性，如果使用了多个 profiler，可以通过此属性区分
  • phase：有 “mount”、”update”、”nested-update”，了解组件是首次挂载还是由于 props、state、hook 触发的重新渲染
  • actualDuration：此次更新的渲染耗时，这个时间可能是优化之后的（使用了memo、useMemo记忆）
  • baseDuration：估算没有任何优化情况下的渲染耗时，然后与 actualDuration 对比

此组件生产环境默认禁用
