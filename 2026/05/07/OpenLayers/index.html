OpenLayers

JavaScript 地图引擎，能在网页上加载、显示和交互各种来源的地图数据与空间信息服务

基础概念

与 OpenLayers 类似的库还有 Leaflet、MapLibre GL JS 和 Cesium，相比于 OpenLayers 的全面，这三者在各自的垂直领域有所优势：

维度 OpenLayers Leaflet MapLibre GL JS Cesium
定位 全功能 GIS 引擎 轻量交互地图 高性能矢量瓦片 3D 地球与时空
渲染 Canvas / WebGL SVG / Canvas WebGL WebGL
2D / 3D 2D 为主 2D 2D / 2.5D 3D 为主
包体积 较大（~500KB+） 小（~40KB） 中等（~200KB） 大（~1MB+）
学习曲线 陡峭 平缓 中等 陡峭
矢量瓦片 支持 需插件 原生核心能力 支持
插件生态 丰富 极丰富 发展中 较少
适用场景 复杂 GIS、OGC 服务 简单展示、标注 自定义底图、大数据量 数字孪生、卫星轨迹

地图 Map

OpenLayers 中的核心组件，从ol/Map模块中导入，通过指定一个渲染容器（dom 选择器）来渲染。所有的地图配置可在初始化的时候指定，也可以后续通过setter方法来指定（比如setTarget()）

视图 View

地图不负责指定地图的中心点、缩放等级和投影，所以需要使用ol/View的实例属性

知识点：

  1. projection（投影）决定了：
  • 地图中心点的坐标系
  • 地图分辨率计算单位（米/度等）

默认值：如果不手动指定投影，默认使用墨卡托投影（EPSG:3857）（Mercator），地图单位米。绝大多数情况下不需要配置投影。

  1. zoom（缩放级别）核心机制，是快速指定地图分辨率的简便方式，缩放层级计算由3个参数决定：
参数名 默认值 作用
maxZoom 28 最大支持的缩放级别
zoomFactor 2 缩放因子（分辨率缩放倍数）
maxResolution 自动计算 让投影的有效范围刚好适配 256×256 像素瓦片

分辨率计算规则：

  • 缩放级别 0（最小）：分辨率=maxResolution（单位/像素）
  • 后续级别每增加 1 级缩放：分辨率=上一级分辨率÷zoomFactor
  • 直到上限（maxZoom）为止

也就是说：

  • 缩放级别越大，地图放得越大，分辨率越高（1 像素代表的实际距离越小）
  • 缩放级别越小，地图缩得越小，分辨率越低（1 像素代表的实际距离越大）
  • 每放大一级，地图细节翻倍（因为除以 2）

数据源 Source

是获取地图数据的核心类，在开发时先建source，放进layer

给一个图层（layer）提供远程数据的时候，使用ol/source/Source子类，这些子类支持：

  • 免费/商用地图瓦片服务（如 OpenStreetMap、高德地图）
  • OGC 标准服务（如 WMS、WMTS）
  • 矢量数据格式（如 GeoJSON、KML）

OGC（Open Geospatial Consortium），开放地理空间信息联盟，相当于是地图的国际标准。只要遵循这个标准，任何地图软件（OpenLayers、高德等）都能互相调数据、兼容。📑详见下方参考篇章。

数据源类型

在 OpenLayers 中，图层（layer）为显示容器，数据源（source）理解为地图内容从哪来

数据源支持以下类型：

  1. 瓦片地图

💡 瓦片地图最为常见

谷歌地图、高德、百度、OpenStreetMap 都是这种

  1. 标准地图服务

💡 企业/政府常用

  1. 矢量数据

💡 点、线、面

图层 Layer

图层 = 把数据源“画”在屏幕上的方式，不同图层对应不同数据类型，不能混用。查看下面类型

图层类型 数据样子 典型用途
Tile 小方块图片 底图（OSM / 高德）
Image 一整张图 WMS 动态图
Vector 点、线、面 标记、轨迹、区划
VectorTile 矢量小方块 高性能矢量底图

ol/layer/Tile（瓦片图层）

💡 最常用，互联网底图基本都用这个，看到的高德地图、OpenStreetMap 底图

作用：显示切成小方块的地图图片（瓦片）

数据源：OSM、高德、谷WMTS、TileWMS

特点：

  • 一张地图切成 256x256 小图片
  • 缩放级别固定，加载快、不卡顿

ol/layer/Image（图片图层）

💡 动态生成的气象图、专题图、非瓦片的 WMS 服务

作用：显示一整张地图图片，不是小方块

数据源：ImageWMS（项服务器请求一整张地图图片）

特点：

  • 服务器直接返回一张完整图片
  • 缩放时会重新请求整张图
  • 加载比瓦片慢，但是支持任意比例尺/范围

使用ImageWMS的情况：

  1. 后端是 WMS 标准地图服务（政府 / 企业 GIS 系统）
  2. 需要动态计算的地图（气象、实时监控、实时统计）
  3. 地图不需要切成瓦片

和瓦片对比，Tile（瓦片）= 拼图（一块一块加载）; ImageWMS = 一张完整的照片（每次都发整张）

ol/layer/Vector（矢量图层）

💡 业务点位、轨迹、区划边界 都用这个

作用：显示点、线、面、坐标数据

数据源：GeoJSON、KML、WFS

特点：

  • 前端渲染，可交互、可点击、可修改样式
  • 数据量大了会卡顿

ol/layer/VectorTile（矢量瓦片图层）

💡 更高级些，现在大厂地图底图都用这个，比如：Mapbox、高德新版矢量底图

作用：显示切成瓦片的矢量数据

数据源：VectorTile

特点：

  • 结合了瓦片的速度 + 矢量的灵活
  • 海量数据不卡顿
  • 样式前端自定义

OGC

OGC = Open Geospatial Consortium，开放地理空间信息联盟。是全球地理地图行业统一制定通用接口标准的组织，相当于地图界的国标 / 国际标准。

只要遵守 OGC 标准，任何地图软件（OpenLayers、ArcGIS、高德、自研平台）都能互相调数据、互相兼容。

OpenLayers 里 TileWMS、WMTS 这些数据源，就是专门用来接入这类标准服务的

常见的OGC服务

  1. WMS（Web Map Service）

网络地图服务

  • 返回：静态图片瓦片/整张地图图片
  • 用图：看底图、专题图
  • 特点：发请求，服务器画好一张图片返回给前端
  1. WMTS（Web Map Tile Service）

网络底图瓦片服务

  • 是WMS的升级版，按固定瓦片、固定图层切好
  • 速度快、性能更高，现在主流项目优先推荐使用
  1. WFS（Web Feature Service）

网络要素服务

  • 返回：矢量原始数据（点、线、面坐标、属性字段）
  • 可以前端高亮、查询、编辑、弹窗，不是图片
  1. WCS（Web Coverage Service）

网络栅格覆盖服务

  • 多用于气象、遥感、高程影像这类栅格数据

与普通瓦片（XYZ）的区别

类型 特点 适用场景
XYZ / 高德 / 谷歌瓦片 厂商自定义规则，不是标准，换平台可能不能直接用 商业地图、快速加载
OGC WMS/WMTS/WFS 国际通用标准，所有 GIS 框架通用 政务、国企、国土项目

GeoJSON

是专门用来存地理空间数据的JSON数据格式，存点、线、面、行政区、轨迹、点位等地理要素

后缀一般为：.geojson，所有地图框架都原生支持。能描述的地理图形：

  1. Point 点，标记、门店、摄像头、兴趣点
  2. LineString 线，道路、轨迹、航线、河流
  3. Polygon 面，城市边界、小区范围、行政区域、地块

结构规则

字段 说明
type 类型（FeatureCollection / Feature / Point / LineString / Polygon…）
coordinates 坐标，格式因 type 而异
properties 自定义属性（名称、地址、人口、类型等）
geometry 几何体，必填，可为 null

经纬度格式强制 [经度，纬度]（lng, lat），用 WGS84 坐标系Turf.js

  1. 面（Polygon，一块区域）
  1. 多种内容组成的合集

Turf

Turf 是专门操作 GeoJSON 的空间分析能力，输入输出全是 GeoJSON（负责算数据）

在 OpenLayers 扮演角色

Turf.js是 Turf 空间分析功能的 JavaScript 库实现，支持浏览器、NodeJS

NPM安装

或者通过 cdn 引用

与GeoJSON

Turf 的所有函数（距离、面积、缓冲、相交、合并等）都要求传入标准 GeoJSON 对象。计算结果（如缓冲区、交点、合并面）一律返回 GeoJSON，可直接喂给 OpenLayers/Leaflet/Mapbox 渲染
