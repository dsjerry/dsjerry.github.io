ReactNative初始化

主流初始化方式有两种：官方脚手架（React Native CLI）和 Expo。

方式一：官方脚手架（React Native CLI）

React Native CLI 保留完整原生工程和最高定制自由度，适合需要精确控制 React Native 版本或深度集成原生模块的场景。Expo 是官方首选方式，但 CLI 依然受支持，没有被放弃。

环境配置

环境配置和开发机器的系统、目标平台都有关。选对应章节看即可。

如果用 Expo，跳过这里，直接看 Expo 章节。

Android 开发环境

所有平台装 Android Studio 都是一样的流程：下载安装，SDK Manager 里补装 Android 15（VanillaIceCream）SDK。

安装向导中勾选 Android SDK、Android SDK Platform、Android Virtual Device。Windows 额外勾选 Hyper-V 或 Intel HAXM。

SDK 配置：

  1. 打开 SDK Manager（macOS/Linux：Android Studio → Settings → Languages & Frameworks → Android SDK；Windows：More Actions → SDK Manager）
  2. SDK Platforms → 勾选 Show Package Details → 展开 Android 15 (VanillaIceCream) → 确认勾选 Android SDK Platform 35 和对应 System Image（M1 Mac 选 ARM 64 v8a）
  3. SDK Tools → 勾选 Show Package Details → 展开 Android SDK Build-Tools → 确认勾选 36.0.0 和 Android SDK Command-line Tools (latest)
  4. Apply 下载安装。

环境变量：

  • macOS — ~/.zshrc：
  • Linux — ~/.bashrc：
  • Windows — 控制面板 → 用户账户 → 更改环境变量 → 新建用户变量 ANDROID_HOME，值为 %LOCALAPPDATA%\Android\Sdk，并将 %LOCALAPPDATA%\Android\Sdk\platform-tools 加入 Path。

Linux 用户还需要单独装 Watchman（参考官方指南），macOS 的 Watchman 已经在前面装好了。

iOS 开发环境

  1. Mac App Store 装 Xcode（最新版本），iOS Simulator 会一并装好
  2. Xcode → Settings → Locations → Command Line Tools 选择最新版本
  3. Xcode 14+ 还需要 Settings → Platforms → + 安装 iOS 版本
  4. 装 CocoaPods：

用 NVM + zsh 的话，把 NVM 初始化代码从 ~/.zshrc 移到 ~/.zshenv，否则 Xcode 可能找不到 Node。

准备 Android 设备

  • 真机：USB 连电脑后启用调试模式（设置 → 关于手机 → 点击构建号七次 → 开发者选项 → 开启 USB 调试）
  • 模拟器：Android Studio → AVD Manager → Create Virtual Device → Phone → VanillaIceCream API Level 35 镜像 → 创建后点击绿色三角

Windows / Linux（仅 Android）

Node、JDK、Android Studio 装法见上方，各平台差异：

平台 Node 安装 JDK 安装 Android Studio 额外勾选
Windows choco install -y nodejs-lts choco install -y microsoft-openjdk17 Hyper-V 或 Intel HAXM
Linux 各发行版包管理器装 Node >= 22 sudo apt install openjdk-17-jdk 无

Windows 记得把 %LOCALAPPDATA%\Android\Sdk\platform-tools 加入 Path。

方式二：Expo

官方推荐。 React Native 官方已将 Expo 作为首选开发方式，提供文件路由和原生模块标准库，EAS 服务覆盖从开发到发布的全流程。

系统要求

  • Node.js LTS 版
  • macOS、Windows（WSL 2）、Linux 都支持

当前是 SDK 55 过渡期，不带 --template 默认建 SDK 54 项目。计划用 Expo Go 的话就 SDK 54，否则用 --template default@sdk-55 建 SDK 55。

开发方式选择

  • Expo Go：学习和尝试用，扫码即用
  • Development Build：将应用打包成带 Expo 开发工具的独立 App，适合正式开发

常用 Expo 命令

命令 说明
npx expo start 启动开发服务器，终端显示二维码
npx expo run:android 构建并运行 Android（自动启动 Metro）
npx expo run:ios 构建并运行 iOS（自动启动 Metro）
npx expo prebuild 生成 android/ 和 ios/ 原生工程（需要原生代码时）
npx expo install <package> 安装 Expo 兼容的包（自动匹配合适版本）
eas build:configure 配置 EAS 构建
eas build --platform android --profile development 创建 Android 开发构建
eas build --platform ios --profile development 创建 iOS 开发构建
eas build --platform android --profile production 创建 Android 生产构建
eas build --platform ios --profile production 创建 iOS 生产构建
eas submit --platform android 提交 Android 构建到 Play Store
eas submit --platform ios 提交 iOS 构建到 App Store
eas update --auto 热更新推送到已安装应用

两种方式对比

对比项 React Native CLI Expo
初始化速度 较慢 快，聚焦 JS 层
环境配置 需手动配置 Android Studio / Xcode Expo Go 扫码即用
原生模块 Expo SDK 封装，够用
热更新/预览 需手动配置 Expo Go 扫码即用
项目体积 较大 较小
版本控制 可精确指定任意 RN 版本 跟随 Expo SDK 版本
自定义原生代码 支持 受限，需 expo prebuild

打包与发布

Expo 打包靠 EAS Build 云端构建 + EAS Submit 自动提交商店。

构建类型

类型 用途 格式
Development Build 团队内部预览，带 Expo 开发工具 APK（Android） / ad-hoc（iOS）
Preview Build 内部分发，无需应用商店审核 APK / ad-hoc
Production Build 提交到 Google Play / App Store AAB（Android） / ipa（iOS）

常见问题

Q：初始化报 jest-haste-map 或 watchman 错误？

A：Linux / Windows 用户可忽略 iOS 相关的 watchman 警告。Windows 建议用 WSL2 开发。

Q：Android 模拟器启动不了？

A：检查 ANDROID_HOME 是否指向正确目录（macOS: ~/Library/Android/sdk，Windows: %LOCALAPPDATA%\Android\Sdk，Linux: ~/Android/Sdk）。

Q：高版本 JDK 导致构建失败？

A：React Native 推荐 JDK 17。不兼容的话，改 android/gradle/wrapper/gradle-wrapper.properties 里的 distributionUrl 升 Gradle 版本，或者降 JDK。

Q：Expo 项目想获得原生能力怎么办？

A：执行 npx expo prebuild 生成原生工程，代码不用动，但原生配置要手动维护。
