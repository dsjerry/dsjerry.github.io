Anthropic 兼容方式

Anthropic（Claude 的开发商）使用另一套协议，与 OpenAI 的区别：

  • API Key 通过 x-api-key 请求头传递（不是 Authorization）
  • 必须显式指定 max_tokens
  • 接口地址是 /v1/messages
  • 返回结构不同，不是 choices[0].message.content，而是 content[0].text

错误处理

常见错误类型及处理方式：

网络错误 — 请求发不出去

Token 超限 — 超过模型限制

deepseek-chat 单次输出上限约 8192 tokens，上下文窗口 128K。max_tokens 过大或对话历史累积太长，会触发 context_length_exceeded 错误。

应对方法：控制 max_tokens 不超过模型上限；对超长对话做截断或总结。

API Key 无效或权限不足 — 401 / 403 错误

生产环境建议加上重试逻辑（指数退避），应对 429（请求过于频繁）错误。

API 汇总

OpenAI 兼容协议通过 Base URL + 路径 组合成完整端点。例如：https://api.deepseek.com/v1 + /chat/completions = https://api.deepseek.com/v1/chat/completions。

通用路径（OpenAI 兼容）

路径 用途
/chat/completions 对话补全（最常用）
/completions 文本补全（旧接口，已基本被 chat 取代）
/embeddings 向量嵌入
/models 可用模型列表
/images/generations 文生图（部分供应商）
/audio/transcriptions 语音转文本（Whisper 系）
/audio/speech 文本转语音（TTS）
/moderations 内容审核
/files / /batches 文件上传与批处理
